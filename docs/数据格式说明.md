# 数据格式说明

## 一、数据流概览

```
Xsens IMU 传感器 (.txt)
        ↓
  data_process.m（四元数去重力 + 滤波）
        ↓
  Excel (.xlsx)（滤波后特征）
        ↓
  1_rename.m → 2_TOmat.m（转为 .mat）
        ↓
  3_pijie_.m（多受试者拼接）
        ↓
  lable.m（力板阈值标注相位）
        ↓
  DataEnhancement.m（数据增强）
        ↓
  tiqu_6.m（提取6维特征）
        ↓
  4_final.m（划分训练/测试集）
        ↓
  STS-PD dataset.mat（最终数据集）
```

---

## 二、原始传感器数据（Xsens .txt）

### 2.1 文件来源
- 设备：Xsens MTw 无线惯性传感器
- 软件：Xsens MT Manager 导出
- 采样率：100 Hz

### 2.2 文件格式
- 编码：纯文本
- 分隔符：Tab（`\t`）
- 前6行为头信息（元数据），第7行起为数据
- 代码加载方式：`importdata(filename, '\t', 6)`

### 2.3 数据列定义

| 列号 | 字段名 | 单位 | 说明 |
|------|--------|------|------|
| 1 | 计数器/时间戳 | - | 帧序号 |
| 2 | Acc_X | m/s² | X轴加速度（传感器坐标系） |
| 3 | Acc_Y | m/s² | Y轴加速度 |
| 4 | Acc_Z | m/s² | Z轴加速度 |
| 5 | Gyr_X | rad/s | X轴角速度 |
| 6 | Gyr_Y | rad/s | Y轴角速度 |
| 7 | Gyr_Z | rad/s | Z轴角速度 |
| 8 | Quat_q0 | - | 四元数实部（w） |
| 9 | Quat_q1 | - | 四元数虚部（x） |
| 10 | Quat_q2 | - | 四元数虚部（y） |
| 11 | Quat_q3 | - | 四元数虚部（z） |

### 2.4 示例数据（前3行）
```
1    9.72    0.15    -1.23    0.02    -0.05    0.01    0.998    0.012    -0.003    0.058
2    9.71    0.16    -1.25    0.03    -0.04    0.02    0.998    0.012    -0.003    0.058
3    9.73    0.14    -1.22    0.02    -0.06    0.01    0.998    0.013    -0.003    0.058
```

## 三、预处理后的中间数据（Excel .xlsx）

### 3.1 处理流程（`data_process.m`）
1. 读取原始 `.txt` 中的加速度（列2-4）和四元数（列8-11）
2. 四元数共轭 → 四元数旋转 → 将加速度转换到世界坐标系
3. 减去重力分量 `[0, 0, 9.8]`，得到线性加速度
4. 12阶低通巴特沃斯滤波（截止频率1.8Hz）
5. 提取角速度Y轴并滤波

### 3.2 输出列定义（4列）

| 列号 | 内容 | 单位 | 说明 |
|------|------|------|------|
| 1 | raw_linAcc_Z | m/s² | 垂直线性加速度（未滤波） |
| 2 | gyr_Y | °/s | Y轴角速度（未滤波） |
| 3 | acc_Filt | m/s² | 垂直线性加速度（滤波后） |
| 4 | gyr_magFilt | °/s | Y轴角速度（滤波后） |

## 四、力板数据（用于标注）

### 4.1 数据来源
- 设备：AMTI BP400600-OP-1000 力板
- 采样率：100 Hz（与IMU同步）
- 2块力板：脚下 + 座椅下

### 4.2 标注规则（`lable.m`）

基于垂直地面反力（GRF）的阈值条件划分5个相位：

| 相位 | 标签 | GRF条件 |
|------|------|---------|
| 初始坐姿 | 1 | GRF ≈ 24%体重，保持恒定 |
| 屈曲动量 | 2 | GRF开始增加 → 座椅力=0N |
| 动量转移 | 3 | 座椅力=0N → GRF达到峰值 |
| 伸展 | 4 | GRF峰值 → GRF≈体重 |
| 稳定站立 | 5 | GRF≈体重，保持恒定 |

## 五、STS-PD 数据集（最终格式）

### 5.1 文件信息
- 文件名：`STS-PD dataset.mat`
- 大小：约 25 MB
- 格式：MATLAB v5 `.mat` 文件

### 5.2 结构体层次

```
STS-PD dataset.mat
└── sts                          % MATLAB struct
    ├── train                    % cell array (1 × N_train)
    │   ├── {1}                  % double matrix [feat_dim × seq_len_1]
    │   ├── {2}                  % double matrix [feat_dim × seq_len_2]
    │   └── ...                  % 每个样本序列长度可能不同
    ├── trainlabels              % double array  [1 × N_train]
    ├── test                     % cell array (1 × N_test)
    │   ├── {1}                  % double matrix [feat_dim × seq_len_1]
    │   ├── {2}                  % double matrix [feat_dim × seq_len_2]
    │   └── ...
    └── testlabels               % double array  [1 × N_test]
```

### 5.3 Python 加载方式

```python
from scipy import io
import numpy as np

data = io.loadmat('STS-PD dataset.mat')

# 访问数据
train_data   = data['sts'][0, 0]['train'].T.squeeze()    # object array of matrices
train_labels = data['sts'][0, 0]['trainlabels'].squeeze()  # 1D int array
test_data    = data['sts'][0, 0]['test'].T.squeeze()
test_labels  = data['sts'][0, 0]['testlabels'].squeeze()

# 单个样本
sample = np.array(train_data[0], dtype=np.float32)  # shape: (feat_dim, seq_len)
label  = train_labels[0]                              # int
```

### 5.4 MATLAB 加载方式

```matlab
load('STS-PD dataset.mat');
% sts.train{1}       → 单个样本矩阵 [feat_dim × seq_len]
% sts.trainlabels(1) → 标签
% sts.test{1}        → 测试样本
% sts.testlabels(1)  → 测试标签
```

### 5.5 实际数据统计

| 属性 | 值 |
|------|-----|
| 训练样本数 | 15,800 |
| 测试样本数 | 6,800 |
| 总样本数 | 22,600 |
| 特征维度 | 2（垂直加速度 + 垂直角速度） |
| 最大时间步 | 128 |
| 类别数 | 5 |
| 标签范围 | 1 ~ 5（MATLAB 1-based） |
| 划分比例 | 约 70% / 30% |

### 5.6 各类样本分布

| 标签 | 相位名称 | 含义 |
|------|----------|------|
| 1 | Initial Sitting | 初始坐姿 |
| 2 | Flexion Momentum | 屈曲动量 |
| 3 | Momentum Transfer | 动量转移 |
| 4 | Extension | 伸展 |
| 5 | Stable Standing | 稳定站立 |

### 5.7 关键注意事项

1. **变长序列**：每个样本的时间步长不同（最长128步），需零填充对齐后才能batch训练
2. **标签偏移**：MATLAB标签从1开始，Python中需减1变为0-based（`labels -= labels.min()`）
3. **特征维度**：虽然原始IMU有12维数据（2个传感器各6维），但最终数据集只保留了2维（垂直加速度+垂直角速度）
4. **数据增强**：数据集已包含增强后的样本（高斯噪声+时间伸缩），总量约22,600

## 六、数据增强方法（`DataEnhancement.m`）

| 方法 | 参数 | 说明 |
|------|------|------|
| 高斯噪声 | SNR = 20dB | 在时间序列上添加白噪声 |
| 时间伸缩 | 0.5x ~ 1.5x | 使用 `interp1` 插值将序列拉伸/压缩 |

目的：解决各相位样本数量不平衡问题，提高模型泛化能力。

## 七、模型输入格式要求

### 7.1 深度学习模型（CNN-BiLSTM-Attention 等）
```
输入张量形状: (batch_size, max_seq_len, feat_dim)
             = (64, 128, 2)
```
- 变长序列零填充至 `max_seq_len=128`
- `feat_dim=2`：垂直加速度 + 垂直角速度

### 7.2 传统机器学习（SVM/KNN/RF 等）
```
输入矩阵形状: (n_samples, max_seq_len × feat_dim)
             = (15800, 256)
```
- 零填充后展平为一维向量
