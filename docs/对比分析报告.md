## 二、数据处理流程对比

### 2.1 论文描述的流程
1. 原始3轴加速度+3轴角速度信号采集（2个IMU，100Hz）
2. 四元数姿态融合，去除重力分量得到线性加速度
3. 12阶低通巴特沃斯滤波（截止频率1.7Hz）
4. 组合加速度+角速度特征作为模型输入

### 2.2 代码实现（`IMU dataprocessing/`）

| 步骤 | 论文 | 代码 | 对比 |
|------|------|------|------|
| 数据读取 | Xsens MTw传感器 | `data_process.m`：读取`.txt`文件，tab分隔，前6行头信息 | ✅ |
| 姿态融合 | 内置传感器融合算法 | `data_process.m`：使用Madgwick AHRS算法（`MadgwickAHRS`类） | ⚠️ 论文说"内置算法"，代码用开源Madgwick |
| 去重力 | 四元数旋转 + 减去[0,0,9.8] | `data_process.m`：`quaternConj`+`quaternProd`实现四元数旋转去重力 | ✅ |
| 滤波 | 12阶巴特沃斯，1.7Hz截止 | `data_process.m`：`butter(12, 2*1.7/SampleFreq)` — **但代码中截止频率归一化为`2*1.7/100=0.034`** | ✅ |
| 信号分割 | 固定滑窗1.28s | `fengge.m`：基于峰值检测分割STS周期 | ⚠️ 分割策略略有不同 |

### 2.3 差异分析
- **姿态融合算法**：论文称使用"IMU内置传感器融合算法计算四元数"，但代码使用的是开源的 Madgwick AHRS 算法。这可能是因为论文简化了描述，或者实际中两者结果接近。
- **信号分割**：`fengge.m` 使用峰值检测来分割每次STS动作，而论文描述使用1.28s固定滑窗。实际流程可能是先用峰值检测分割出每个STS动作周期，再用固定滑窗在周期内分割各相位。

## 六、数据集构建流程对比

### 论文描述流程
1. 15名受试者，每人5次STS
2. 力板+IMU同步采集（100Hz）
3. 固定滑窗分割（1.28s）
4. 手动标注+数据增强
5. 27,276样本，70/30划分

### 代码实现流程
1. `1_rename.m` → 批量重命名原始Excel文件
2. `2_TOmat.m` → Excel转.mat格式
3. `3_pijie_.m` → 多受试者数据拼接
4. `lable.m` → 基于力板阈值自动标注相位
5. `DataEnhancement.m` → 数据增强（加噪声+时间伸缩）
6. `tiqu_6.m` → 从12维提取6维特征
7. `4_final.m` → 按7:3划分生成最终STS-PD数据集

### 差异
- 论文称"手动标注"花费约15小时，但代码中 `lable.m` 实现的是**自动阈值标注**。推测实际流程是自动标注后人工校验。
- 数据增强方法（高斯噪声+时间伸缩）在论文中仅简略提及，代码中有完整实现。

## 八、代码质量与可复现性评估

### 优点
1. **方法覆盖全面**：论文中提到的所有对比方法均有代码实现
2. **模型架构忠实**：CNN-BiLSTM-Attention的实现与论文描述高度一致
3. **数据集公开**：提供了完整的STS-PD dataset.mat
4. **流程完整**：从原始数据处理到最终模型评估的完整pipeline

### 不足/可改进之处
1. **硬编码路径**：所有代码使用绝对路径（如`C:\Users\Administrator\Desktop\cx\...`），不利于他人复现
2. **缺少依赖说明**：无`requirements.txt`或环境配置文件
3. **代码注释**：部分中文注释，部分无注释，可读性参差不齐
4. **缺少评估脚本整合**：各模型分散在不同文件，没有统一的评估对比脚本
5. **可视化代码**：引用了`polt`模块（可能是自定义模块），但未包含在仓库中
6. **MATLAB工具箱依赖**：`data_process.m`依赖`quaternion_library`和`ximu_matlab_library`，仓库中未提供

## 九、总结

### 一致性评分：★★★★☆（4/5）

项目代码与论文描述**高度一致**，核心模型架构、训练参数、对比方法、数据处理流程均忠实实现了论文内容。主要差异在于：

1. 姿态融合算法的描述差异（论文简化描述 vs 代码用Madgwick）
2. 标注方式的描述差异（论文称手动 vs 代码是半自动）
3. 工程规范性不足（硬编码路径、缺少依赖管理）

**整体结论**：该仓库是论文的**有效支撑代码**，包含了复现论文结果所需的全部核心算法实现和数据集，但在工程规范性和可复现便利性方面有改进空间。
