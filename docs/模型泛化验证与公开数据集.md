# 模型泛化验证与公开数据集

## 一、为什么需要泛化验证

论文提出的 CNN-BiLSTM-Attention 模型在自定义 STS-PD 数据集上取得了 99.5% 的准确率。为了证明模型不仅在特定数据集上表现好，还具有**通用的时间序列分类能力**，论文在3个公开的人体活动识别（HAR）数据集上进行了泛化验证（TABLE IV）。

这些公开数据集覆盖了不同的：
- 传感器类型（智能手机 vs 可穿戴传感器）
- 活动类型（日常活动、手势动作）
- 类别数量（6类 ~ 18类）
- 数据规模和平衡性

---

## 二、公开数据集概览

| 数据集 | 活动类型 | 类别数 | 传感器 | 受试者 | 采样率 | 特点 |
|--------|----------|--------|--------|--------|--------|------|
| UCI-HAR | 6种日常活动 | 6 | 智能手机（加速度+陀螺仪） | 30人 | 50Hz | 最经典的HAR基准 |
| WISDM | 6种日常活动 | 6 | 智能手机加速度计 | 36人 | 20Hz | 数据不平衡 |
| OPPORTUNITY | 手势+日常动作 | 18 | 全身可穿戴传感器 | 4人 | 30Hz | 高维特征、严重不平衡 |

## 三、UCI-HAR 数据集

### 3.1 基本信息
- **全称**：University of California Irvine - Human Activity Recognition Using Smartphones
- **论文引用**：Anguita et al., 2012 [25]
- **下载地址**：https://archive.ics.uci.edu/ml/datasets/human+activity+recognition+using+smartphones
- **备用下载**：https://www.kaggle.com/datasets/uciml/human-activity-recognition-with-smartphones

### 3.2 数据详情
- **受试者**：30名志愿者（19-48岁）
- **传感器**：Samsung Galaxy S II 智能手机（腰部佩戴）
- **采样率**：50 Hz
- **信号**：3轴加速度 + 3轴陀螺仪
- **特征**：已提取561维时频域特征（也提供原始信号）

### 3.3 活动类别（6类）
| 标签 | 活动 | 中文 |
|------|------|------|
| 1 | WALKING | 行走 |
| 2 | WALKING_UPSTAIRS | 上楼梯 |
| 3 | WALKING_DOWNSTAIRS | 下楼梯 |
| 4 | SITTING | 坐 |
| 5 | STANDING | 站 |
| 6 | LAYING | 躺 |

### 3.4 数据划分
- 训练集：7,352 样本（21人）
- 测试集：2,947 样本（9人）
- 滑动窗口：2.56秒（128个采样点），50%重叠

### 3.5 使用方式
```python
# 下载后目录结构
# UCI HAR Dataset/
# ├── train/
# │   ├── X_train.txt        # (7352, 561) 特征向量
# │   ├── y_train.txt        # (7352,) 标签
# │   └── Inertial Signals/  # 原始信号（9个文件，每个128×7352）
# └── test/
#     ├── X_test.txt
#     ├── y_test.txt
#     └── Inertial Signals/

import numpy as np

# 方式1：使用预提取的561维特征
X_train = np.loadtxt('UCI HAR Dataset/train/X_train.txt')  # (7352, 561)
y_train = np.loadtxt('UCI HAR Dataset/train/y_train.txt')  # (7352,)

# 方式2：使用原始信号（适合深度学习）
# 读取9个通道的原始数据，拼接为 (samples, 128, 9)
channels = ['body_acc_x', 'body_acc_y', 'body_acc_z',
            'body_gyro_x', 'body_gyro_y', 'body_gyro_z',
            'total_acc_x', 'total_acc_y', 'total_acc_z']
X_raw = []
for ch in channels:
    data = np.loadtxt(f'UCI HAR Dataset/train/Inertial Signals/{ch}_train.txt')
    X_raw.append(data)
X_raw = np.stack(X_raw, axis=-1)  # (7352, 128, 9)
```

### 3.6 论文中的结果（TABLE IV）
CNN-BiLSTM-Attention 在 UCI-HAR 上的表现应优于大多数对比方法。

## 四、WISDM 数据集

### 4.1 基本信息
- **全称**：Wireless Sensor Data Mining - Activity Recognition
- **论文引用**：Kwapisz et al., 2011 [26]
- **下载地址**：https://www.cis.fordham.edu/wisdm/dataset.php
- **备用下载**：https://www.kaggle.com/datasets/anshtanwar/wisdm-activity-dataset

### 4.2 数据详情
- **受试者**：36名志愿者
- **传感器**：Android 智能手机加速度计（裤袋中）
- **采样率**：20 Hz
- **信号**：仅3轴加速度

### 4.3 活动类别（6类）
| 标签 | 活动 | 中文 | 样本占比 |
|------|------|------|---------|
| 0 | Walking | 行走 | 38.6% |
| 1 | Jogging | 慢跑 | 31.2% |
| 2 | Upstairs | 上楼梯 | 11.2% |
| 3 | Downstairs | 下楼梯 | 9.1% |
| 4 | Sitting | 坐 | 5.5% |
| 5 | Standing | 站 | 4.4% |

### 4.4 数据特点
- ⚠️ **严重不平衡**：Walking 和 Jogging 占比近70%，Sitting 和 Standing 不到10%
- 需要使用 F1-score（而非仅准确率）评估模型性能
- 原始数据为连续时间序列，需自行滑窗分割

### 4.5 使用方式
```python
import pandas as pd
import numpy as np

# 原始数据格式：user_id, activity, timestamp, x, y, z
df = pd.read_csv('WISDM_ar_v1.1_raw.txt', 
                  header=None, 
                  names=['user', 'activity', 'timestamp', 'x', 'y', 'z'],
                  comment=';')

# 滑窗分割（窗口大小200，步长100）
window_size = 200
step_size = 100
segments = []
labels = []

for activity in df['activity'].unique():
    subset = df[df['activity'] == activity]
    for start in range(0, len(subset) - window_size, step_size):
        segment = subset.iloc[start:start+window_size][['x', 'y', 'z']].values
        segments.append(segment)
        labels.append(activity)

X = np.array(segments)  # (n_samples, 200, 3)
y = np.array(labels)
```

## 五、OPPORTUNITY 数据集

### 5.1 基本信息
- **全称**：OPPORTUNITY Activity Recognition
- **论文引用**：Roggen et al., 2012 [27]
- **下载地址**：https://archive.ics.uci.edu/ml/datasets/opportunity+activity+recognition
- **挑战赛**：https://opportunity-project.eu/

### 5.2 数据详情
- **受试者**：4人
- **传感器**：全身72个传感器（包括身体、物体和环境传感器）
- **采样率**：30 Hz
- **信号维度**：113维（去除缺失列后）

### 5.3 活动类别
包含多个粒度的标注，常用的手势识别任务包含**18类**手势动作：

| 类型 | 示例 |
|------|------|
| 手势动作 | Open Door 1/2, Close Door 1/2, Open Fridge, Close Fridge |
| 抽屉操作 | Open Dishwasher, Close Dishwasher, Open Drawer 1/2/3, Close Drawer 1/2/3 |
| 清洁动作 | Clean Table, Drink from Cup, Toggle Switch |
| 空活动 | Null（无特定动作） |

### 5.4 数据特点
- ⚠️ **极度不平衡**：Null 类占比超过70%
- ⚠️ **高维数据**：113维传感器特征
- ⚠️ **缺失值**：部分传感器通道存在NaN
- 通常需要对 Null 类进行下采样或使用加权损失函数

### 5.5 使用方式

# 提取特征和标签

## 六、泛化验证实验方案

### 6.1 实验目标
验证 CNN-BiLSTM-Attention 模型在不同数据集、不同活动类型、不同传感器配置下的通用性。

### 6.2 统一评估指标
| 指标 | 计算方式 | 说明 |
|------|----------|------|
| Accuracy | 正确数/总数 | 整体准确率 |
| Precision | TP/(TP+FP) | 精确率（加权平均） |
| Recall | TP/(TP+FN) | 召回率（加权平均） |
| F1-Score | 2×P×R/(P+R) | F1分数（加权平均，不平衡数据的核心指标） |

### 6.3 适配步骤

对每个公开数据集，需要：

1. **下载数据**
2. **预处理**：滑窗分割（如需要）、缺失值处理、标准化
3. **调整模型参数**：
   - `input_dim`：匹配特征维度（UCI=9, WISDM=3, OPP=113）
   - `num_classes`：匹配类别数（UCI=6, WISDM=6, OPP=18）
4. **训练和评估**

### 6.4 使用加强版脚本

需要编写数据加载适配器，将各数据集转换为统一格式 `(X_train, y_train, X_test, y_test)`，然后复用 `enhance/train.py` 的训练逻辑：

```python
# 示例：修改 train.py 的 load_data 函数以支持不同数据集
python train.py --data uci_har --model cnn_bilstm_attention --epochs 100
python train.py --data wisdm --model cnn_bilstm_attention --epochs 100
python train.py --data opportunity --model cnn_bilstm_attention --epochs 100
```

## 八、不平衡数据处理建议

对于 WISDM 和 OPPORTUNITY 数据集的类别不平衡问题：

| 方法 | 说明 | 适用场景 |
|------|------|---------|
| 加权交叉熵 | `nn.CrossEntropyLoss(weight=class_weights)` | 最简单，推荐首选 |
| 过采样（SMOTE） | 对少数类样本进行合成过采样 | 传统ML方法 |
| 欠采样 | 对多数类随机丢弃 | 数据量充足时 |
| 数据增强 | 对少数类添加噪声/时间伸缩 | 本项目已有实现 |
| Focal Loss | 降低易分类样本权重 | 极度不平衡时 |

```python
# 加权交叉熵示例
from collections import Counter
counts = Counter(y_train)
total = sum(counts.values())
weights = torch.FloatTensor([total / counts[i] for i in range(num_classes)])
weights = weights / weights.sum() * num_classes  # 归一化
criterion = nn.CrossEntropyLoss(weight=weights.to(device))
```

## 九、总结

| 数据集 | 难度 | 类别 | 核心挑战 | 下载 |
|--------|------|------|---------|------|
| **STS-PD** | ⭐⭐ | 5 | 变长序列 | 仓库自带 |

泛化验证是证明模型通用性的关键步骤。建议按 UCI-HAR → WISDM → OPPORTUNITY 的顺序逐步增加难度进行验证。
